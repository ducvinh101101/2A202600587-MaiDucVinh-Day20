3. Quality Formula

Quality is scored with a weighted research rubric. The current lab uses deterministic proxy scores; a formal study can replace them with human or LLM-judge scores.

Quality = 0.25 * Correctness
        + 0.20 * Source Grounding
        + 0.20 * Completeness
        + 0.15 * Research Judgment
        + 0.10 * Clarity
        + 0.10 * Constraint Following

4. Metric Calculation Details

This section explains the benchmark code in evaluation/benchmark.py. The benchmark runs a system, reads the final ResearchState, then computes latency, token usage, estimated cost, citation coverage, errors, and quality.

Overall benchmark flow

started = perf_counter()
state = runner(query)
latency = perf_counter() - started
input_tokens, output_tokens = _token_usage(state)
citation_coverage = _citation_coverage(state)
error_rate = 1.0 if state.errors else 0.0
quality, quality_breakdown = _quality_score(state, citation_coverage, error_rate)
estimated_cost = _estimate_gemini_flash_cost(input_tokens, output_tokens)

Token usage

Token usage is summed from every AgentResult.metadata. Baseline usually has one LLM result. Multi-agent has multiple LLM results from Researcher, Analyst, and Writer.

input_tokens = sum(result.metadata["input_tokens"])
output_tokens = sum(result.metadata["output_tokens"])

Estimated Gemini cost

The report uses a fixed cost assumption for Gemini Flash-family models, so runs can be compared reproducibly. This is an estimate, not exact provider billing.

cost = input_tokens / 1,000,000 * 0.10
     + output_tokens / 1,000,000 * 0.40

Citation coverage

Citation coverage checks whether the final answer cites retrieved sources with markers such as [1], [2], and so on.

citation_coverage = cited_sources / total_sources

If there is no final answer or no retrieved sources, citation coverage is 0.0.

Error count

Errors are counted from state.errors. A run with at least one error gets error_rate = 1.0; otherwise it gets 0.0.

errors = len(state.errors)
error_rate = 1.0 if state.errors else 0.0

Rubric component formulas

ComponentWeightFormula used in code
correctness25%0 if no final answer. Otherwise starts at 7. +1 if sources exist. +1 if both research_notes and analysis_notes exist. -4 if the run has errors. Clamped to 0-10.
source_grounding20%min(citation_coverage * 10, 10).
completeness20%0 if no final answer. Otherwise starts at 4. +2 if answer length >= 600. +1 if length >= 1200. +1 each for research_notes, analysis_notes, and sources. Capped at 10.
research_judgment15%Starts at 4. Adds up to 6 points for signals such as trade-off, uncertainty, limitation, evidence gap, failure, risk, benchmark, caveat, han che, or rui ro.
clarity10%0 if no final answer. Otherwise starts at 5. +1 if answer has line breaks. +2 if it has markdown/list markers. +2 if length is between 500 and 6000 chars. Capped at 10.
constraint_following10%0 if no final answer. Otherwise starts at 7. +1 if prompt asks for output format and answer has expected section markers. +1 if prompt says must include and answer length >= 1000. -2 if there are errors.

Final quality formula

Quality = 0.25 * correctness
        + 0.20 * source_grounding
        + 0.20 * completeness
        + 0.15 * research_judgment
        + 0.10 * clarity
        + 0.10 * constraint_following

Example

For correctness:7.0, source_grounding:0.0, completeness:7.0, research_judgment:4.0, clarity:10.0, and constraint_following:7.0:

Quality = 0.25*7 + 0.20*0 + 0.20*7 + 0.15*4 + 0.10*10 + 0.10*7
        = 1.75 + 0 + 1.4 + 0.6 + 1.0 + 0.7
        = 5.45 ~= 5.5

9. Failure Handling

Gemini can return temporary 503/disconnect errors. The LLM client retries and wraps provider errors; agents fall back to available state instead of crashing the workflow. Tavily search also has a local fallback.
